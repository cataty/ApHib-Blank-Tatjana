Cocktails and Beverages Database

Description

This API allows you to manage a database of cocktails and beverages. It supports CRUD operations (Create, Read, Update, and Delete) on cocktail and beverage resources. The API is built using Node.js and Express, and uses MongoDB as its database.

Cocktails Endpoints

  • GET /api/cocktails - Fetches a list of all the cocktails from the database. de datos.
  • POST /api/cocktails - Adds a new cocktail to the databe. Example of JSON structure: 
                            {
                            "iba": true,
                            "name": "Horse's Neck",
                            "colors": "#D9A031",
                            "glass": "highball",
                            "category": "Longdrink",
                            "ingredients": [
                              { "unit": "cl", "amount": 4, "ingredient": "Cognac" },
                              { "unit": "cl", "amount": 12, "ingredient": "Ginger Ale" },
                              { "special": "Dash of Angostura bitters (optional)" }
                            ],
                            "garnish": "Lemon twist",
                            "preparation": "Build into highball glass with ice cubes. Stir gently. If required, add dashes of Angostura bitters."
                        }
  • GET /api/cocktails/:id - Fetched a cocktail by ID
  • PUT /api/cocktails/:id - Modifies an existing cocktail in the database. Example of JSON structure (fields can be changed one by one): 
                            {
                            "iba": true,
                            "name": "Horse's Neck",
                            "colors": "#D9A031",
                            "glass": "highball",
                            "category": "Longdrink",
                            "ingredients": [
                              { "unit": "cl", "amount": 4, "ingredient": "Cognac" },
                              { "unit": "cl", "amount": 12, "ingredient": "Ginger Ale" },
                              { "special": "Dash of Angostura bitters (optional)" }
                            ],
                            "garnish": "Lemon twist",
                            "preparation": "Build into highball glass with ice cubes. Stir gently. If required, add dashes of Angostura bitters."
                        }
  • DELETE /api/cocktails/:id - Deletes a cocktails from the databse by ID.
  • GET /api/cocktails/categories/:category - Fetches cocktails from the database filtered by category. The category names in the URL are written in lowercase, with words separated by a hyphen. The available category names are: All Day Cocktail, Sparkling Cocktail, Before Dinner Cocktail, After Dinner Cocktail.

  • GET /api/cocktails/glasses/:glass - Fetches cocktails from the database filtered by glass type. The types of glasses are: martini, highball, old-fashioned, margarita, hurricane, champagne-flute.
  • GET api/cocktails/search/name - Searches a cocktail in the database by name. Spaces are replaced by hyphens. Format example: api/cocktails/search/name?name=dry-martini

Beverages Endpoints

  • GET /api/beverages - Fetches a list of all the beverages from the database.
  • POST /api/beverages - Adds a new beverage to the databe. Example of JSON structure: 
                            {
                            "name":"Vodka",
                            "category":"Liquors",
                            "alcoholic":true,
                            "alcoholContent":40,
                        }
  • GET /api/beverages/:id - Fetches a beverage by ID.
  • PUT /api/beverages/:id - Modifies an existing beverage in the database. Example of JSON structure (fields can be changed one by one): 
                            {
                            _id":"681d004699af07746c1cb9da",
                            "name":"Vodka",
                            "category":"Liquors",
                            "alcoholic":true,
                            "alcoholContent":40,
                        }
  • DELETE /api/beverages/:id - Deletes a beverage from the databse by ID.

Users Endpoints

  • GET /api/users - Fetches a list of all the users from the database.
  • POST /api/users - Adds a new user to the databe. Example of JSON structure: 
                            {
                            "name":"Sebastian",
                            "email":"sebastian@davinci.edu.ar",
                            "password":"12345678",
                        }
  • GET /api/users/:id - Fetches a user by ID.
  • PUT /api/users/:id - Modifies an existing user in the database. Example of JSON structure (fields can be changed one by one): 
                            {
                            "_id":"681c7444fae49d65839fe219",
                            "name":"Sebastian",
                            "email":"sebastian@davinci.edu.ar",
                            "password":"12345678",
                        }
  • DELETE /api/users/:id - Deletes a user from the databse by its ID.
