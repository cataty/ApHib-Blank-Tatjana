Cocktail Database

Cocktails Endpoints

  • GET /api/cocktails - Trae todos los cocktails de la base de datos.
  • POST /api/cocktails - Agrega un nuevo cocktail a la base de datos. Ejemplo de estructura: 
                            {
                            "iba": true,
                            "name": "Horse's Neck",
                            "colors": "#D9A031",
                            "glass": "highball",
                            "category": "Longdrink",
                            "ingredients": [
                              { "unit": "cl", "amount": 4, "ingredient": "Cognac" },
                              { "unit": "cl", "amount": 12, "ingredient": "Ginger Ale" },
                              { "special": "Dash of Angostura bitters (optional)" }
                            ],
                            "garnish": "Lemon twist",
                            "preparation": "Build into highball glass with ice cubes. Stir gently. If required, add dashes of Angostura bitters."
                        }
  • GET /api/cocktails/:id - Trae un cocktail por ID.
  • PUT /api/cocktails/:id - Modifica un cocktail en la base de datos por ID. Ejemplo de estructura (no es necesario cambiar todos los campos a la vez) 
                            {
                            "iba": true,
                            "name": "Horse's Neck",
                            "colors": "#D9A031",
                            "glass": "highball",
                            "category": "Longdrink",
                            "ingredients": [
                              { "unit": "cl", "amount": 4, "ingredient": "Cognac" },
                              { "unit": "cl", "amount": 12, "ingredient": "Ginger Ale" },
                              { "special": "Dash of Angostura bitters (optional)" }
                            ],
                            "garnish": "Lemon twist",
                            "preparation": "Build into highball glass with ice cubes. Stir gently. If required, add dashes of Angostura bitters."
                        }
  • DELETE /api/cocktails/:id - Borra un cocktail de la base de datos por ID.
  • GET /api/cocktails/categories/:category - Trae los cocktails de la base de datos, filtrados por categoria. Los nombres de las categorias en la URL se escriben en minusculas, las palabras se separan con un guion.

Beverages Endpoints

  • GET /api/beverages - Trae todas las bebidas de la base de datos.
  • POST /api/beverages - Agrega una nueva bebida a la base de datos. Ejemplo de estructura del Objeto JSON: 
                            {
                            "name":"Vodka",
                            "category":"Liquors",
                            "alcoholic":true,
                            "alcoholContent":40,
                        }
  • GET /api/beverages/:id - Trae una bebida por ID.
  • PUT /api/beverages/:id - Modifica una bebida en la base de datos por ID. Ejemplo de estructura del Objeto JSON: (no es necesario cambiar todos los campos a la vez) 
                            {
                            _id":"681d004699af07746c1cb9da",
                            "name":"Vodka",
                            "category":"Liquors",
                            "alcoholic":true,
                            "alcoholContent":40,
                        }
  • DELETE /api/beverages/:id - Borra una bebida de la base de datos por ID.

Users Endpoints

  • GET /api/users - Trae todos los usuarios de la base de datos.
  • POST /api/users - Agrega un nuevo usuario a la base de datos. Ejemplo de estructura del Objeto JSON: 
                            {
                            "name":"Sebastian",
                            "email":"sebastian@davinci.edu.ar",
                            "password":"12345678",
                        }
  • GET /api/users/:id - Trae un usuario por ID.
  • PUT /api/users/:id - Modifica un usuario en la base de datos por ID. Ejemplo de estructura del Objeto JSON: (no es necesario cambiar todos los campos a la vez) 
                            {
                            "_id":"681c7444fae49d65839fe219",
                            "name":"Sebastian",
                            "email":"sebastian@davinci.edu.ar",
                            "password":"12345678",
                        }
  • DELETE /api/users/:id - Borra un usuario de la base de datos por ID.
